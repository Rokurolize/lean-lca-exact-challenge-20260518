# Stable Projection Audit CLI Frontier

The external audit handoff should treat `python3 audit/external_audit.py --root . --self-check-only` as a valid shorthand for the repository's default `terminal_outcome/terminal_outcome.json`. Requiring callers to repeat that path in self-check mode creates a CLI discrepancy rather than a stronger audit gate; the explicit argument remains available for fixture roots and packet layouts.

The w73-style syntactic guard and the w77 Lean projection shape cover different review layers. The syntactic guard rejects product-success packets whose stable infinity-category evidence is only an ordinary nerve, a comment/string marker, or a stable certificate name without field evidence. The Lean projection shape states the semantic frontier: a future `Dbounded` stable certificate must expose separate finite-limit, finite-colimit, suspension/loop, and pushout/pullback projections from one certificate object.

Product review should require both layers before accepting stable bounded derived infinity-category evidence. The external audit can reject marker-only claims cheaply, but it should not certify stability until the Lean-facing projection fields are connected to real certificate terms rather than names or placeholders. This note does not claim current stable infinity-category evidence.
